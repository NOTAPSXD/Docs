 Security & Limits

Auth & Rate Limits

Everything you need to know about authenticating requests and respecting usage limits.

Authorization

All protected API requests require your bot's API Token sent in the Authorization HTTP header. You can generate or view your token from your bot's edit page under API Integrations (available after bot approval).

HTTP Header
Authorization: your_api_token_here

Tokens are per-bot — each bot has its own API token. Do not share tokens between bots or expose them in client-side code.

IP Whitelisting (Optional) For enhanced security, whitelist up to 5 IPv4/IPv6 addresses on the bot edit page. Requests with a valid token from an unrecognised IP will receive 403 Forbidden. Whitelisting is disabled by default — a value of 0.0.0.0 means all IPs are allowed.

Token Lifecycle

  • Tokens are generated after your bot is approved by the botliy team.
  • You can regenerate your token at any time from the bot edit page — old tokens are invalidated immediately.
  • Tokens do not expire unless regenerated or your bot is removed.

Rate Limits

To maintain system stability, botliy applies global IP-based rate limiting across all API endpoints.

  • Global limit: 1 request per second, per IP address.
  • For stat-posting, use a fixed interval — every 15–30 minutes is recommended.
  • Do not trigger stat posts on every guild join or leave event.
Scope Limit Recommendation
Global (all endpoints) 1 req/s per IP Add delays between requests
POST /stats 1 req/s per IP Post every 15–30 min max
Exceeding Rate Limits Requests that exceed the rate limit receive 429 Too Many Requests. Repeated violations may result in a temporary IP block.

Error Codes

All errors return a JSON body with an error field. HTTP status codes are standard.

200Request succeeded.
400Bad request — malformed JSON or missing required fields.
401Unauthorized — missing or invalid Authorization token.
403Forbidden — valid token but IP is not whitelisted.
404Not found — bot ID does not exist or is not approved.
429Too many requests — rate limit exceeded.
500Internal server error — something went wrong on our end.
