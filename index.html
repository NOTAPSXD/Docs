botliy Developer API

The botliy API lets you integrate your Discord bot with the platform — post live server stats, retrieve vote history, check individual votes, and receive real-time webhook events the moment someone votes. All responses are JSON.

Base URL https://botliy.online
Format: JSON
Auth: Bearer Token
Rate Limit: 1 req/s per IP
Need help? Reach out in our Discord server's #api-support channel. We typically respond within a few hours.

Quick Start

Get up and running in three steps.

1
Submit & get approved
Add your bot at botliy.online/bots/new. Once approved, your API token is available in the bot's edit page under API Integrations.
2
Add the Authorization header
Include your token on every protected request as Authorization: your_token. See Authentication →
3
Post stats & set up a webhook
Use a 30-minute interval to POST /api/bots/:id/stats and configure a webhook URL on your bot's edit page to receive live vote events.

Key Concepts

Before diving in, here are the fundamental ideas behind the botliy API:

  • Bot ID — Your bot's Discord application/client snowflake ID. Used as the :id path parameter in all endpoints.
  • API Token — A secret token generated per bot, used to authenticate write requests. Never expose this in client-side code.
  • Webhook Secret — A separate secret used to verify incoming vote event payloads from botliy's servers.
  • Votes — Users can vote for your bot once every 12 hours. The API tracks and exposes this data in real time.
Keep your tokens secret Never commit your API Token or Webhook Secret to source control. Use environment variables or a secrets manager in production.
