Webhook Integration

Receive instant HTTP notifications when users vote for your bot — no polling required.

Overview

Instead of polling GET /votes to detect new votes, configure a webhook URL on your bot's edit page. botliy will deliver an HTTP POST request to your server the moment a vote is cast — in real time.

  • Webhooks are delivered within seconds of a vote being cast.
  • Your endpoint must be publicly accessible over HTTPS.
  • Respond with any 2xx status to acknowledge receipt.
  • Responses that take longer than 10 seconds will time out.
Webhooks vs. Polling Webhooks are the recommended way to handle vote events. Polling the votes endpoint is less efficient and subject to rate limits.

Setup

1
Create an HTTP endpoint
Your server needs a route that accepts POST requests with a JSON body. It must be reachable from the public internet over HTTPS.
2
Register your webhook URL
Go to your bot's edit page on botliy, paste your endpoint URL into the Webhook URL field, and save.
3
Copy your Webhook Secret
A Webhook Secret is generated alongside your URL. Store it securely — you'll use it to verify incoming requests.
4
Verify & handle events
Compare the incoming Authorization header to your secret. If they match, process the event and respond with 200 OK.

Payload Schema

INCOMING POST your-configured-webhook-url

botliy sends JSON to your endpoint. Verify the Authorization header matches your webhook secret before processing.

Headers Delivered to Your Endpoint
HTTP Headers
Content-Type: application/json Authorization: your_webhook_secret_here
JSON Payload
JSON
{ "bot": "123456789012345678", // Your bot's Discord ID "user": "876543210987654321", // Voter's Discord ID "type": "upvote" // Event type }
Payload Fields
FieldTypeDescription
bot string The Discord snowflake ID of your bot.
user string The Discord snowflake ID of the voter.
type string The event type — always "upvote" currently.

Verification

Always verify that incoming webhook requests actually come from botliy. Do this by comparing the Authorization header to your stored Webhook Secret.

Node.js verification example
app.post('/botliy/webhook', (req, res) => { const secret = process.env.WEBHOOK_SECRET; // Reject requests without the correct secret if (req.headers.authorization !== secret) { return res.status(401).json({ error: 'Unauthorized' }); } // Safe to process the event const { bot, user, type } = req.body; console.log(`User ${user} voted for bot ${bot}`); res.status(200).json({ status: 'ok' }); });
Never skip verification Without checking the Authorization header, anyone could send fake vote events to your endpoint.

Retry Policy

If your endpoint fails to respond with a 2xx status within 10 seconds, botliy will retry the delivery:

  • Attempt 1 — immediately on vote
  • Attempt 2 — 1 minute after first failure
  • Attempt 3 — 5 minutes after second failure

After 3 failed attempts, the event is dropped. Make your webhook handler idempotent — use the user + timestamp to deduplicate if retries are a concern.
